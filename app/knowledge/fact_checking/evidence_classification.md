# Evidence Classification and Factual Relationship Establishment Guide

## Purpose

This guide provides comprehensive standards for:
1. Classifying evidence materials by legal relationship type
2. Establishing factual relationships based on evidence
3. Applying standardized formats for each evidence type
4. Analyzing evidence hierarchy and relationships

## Part 1: Evidence Classification Standards

### Core Principle: Evidence vs. Declaration Materials

**🚨 CRITICAL DISTINCTION**:

**Evidence Materials** (可作为查明依据):
- Contracts, supplements, amendments
- Court judgments, mediation agreements, arbitration awards
- Invoices, receipts, bank transfer records
- Delivery slips, engineering documents
- Confirmation letters, reconciliation statements
- Any objective documents proving facts

**Declaration Materials** (仅作为思考线索):
- Debt declaration forms (债权申报表)
- Debt declaration letters (债权申报书)
- Creditor's unilateral statements
- **NEVER use as fact-finding basis**

### Nine Legal Relationship Types

#### 1. 借款合同 (Loan Contracts)
**Key Identifiers**: Loan agreements, IOU notes, borrowing contracts
**Common Evidence**: Loan contract + bank transfer records
**Core Elements**: Principal, interest rate, term, repayment method

#### 2. 买卖合同 (Sales/Supply Contracts)
**Key Identifiers**: Purchase agreements, sales contracts, supply agreements
**Common Evidence**: Contract + invoices + delivery slips
**Core Elements**: Goods, price, delivery, payment terms

#### 3. 建设工程合同 (Construction Contracts)
**Key Identifiers**: Construction contracts, project agreements
**Common Evidence**: Contract + engineering documents + settlement documents
**Core Elements**: Project scope, contract price, payment terms, quality retention

#### 4. 担保合同 (Guarantee Contracts)
**Key Identifiers**: Guarantee agreements, mortgage contracts, pledge contracts
**Common Evidence**: Guarantee contract + underlying debt contract
**Core Elements**: Guarantee type, scope, guarantor

#### 5. 劳动关系 (Labor Relations)
**Key Identifiers**: Employment contracts, labor agreements
**Common Evidence**: Employment contract + payroll records
**Core Elements**: Wages, benefits, compensation

#### 6. 侵权责任 (Tort Liability)
**Key Identifiers**: Claims based on tortious acts
**Common Evidence**: Usually established by court judgment
**Core Elements**: Tortious act, damages, causation

#### 7. 票据关系 (Negotiable Instruments)
**Key Identifiers**: Bills of exchange, promissory notes, checks
**Common Evidence**: Original bill instrument
**Core Elements**: Amount, maturity date, payee

#### 8. 生效法律文书 (Effective Legal Documents)
**Key Identifiers**: Judgments, mediation agreements, arbitration awards
**Common Evidence**: Effective legal document itself
**Core Elements**: Judgment/award content, effective date

#### 9. 其他 (Others)
**Examples**: Unjust enrichment, negotiorum gestio, etc.

### Independent Debt Relationship Determination

**Rule**: Each independent contract/legal document = One independent debt relationship

**Examples**:
- 6 separate sales contracts = 6 debt relationships (NOT 1)
- 1 contract + 1 supplement = Still 1 relationship (supplement modifies original)
- 1 contract + 1 judgment confirming it = Still 1 relationship (judgment doesn't create new relationship)

### ⚠️ MANDATORY: Evidence Citation Format Standards

**核心原则: 每个事实必须有明确的证据出处**

所有在报告中陈述的事实,必须能够追溯到具体的证据文件和页码。禁止使用模糊引用或无源陈述。

---

#### 标准引用格式 (Standard Citation Format)

**格式1: 合同条款引用** (Contract Clause Citation)
```
[事实陈述]（见《[合同名称]》第X条第X款,证据材料第X页）

示例:
"借款期限为12个月"（见《借款合同》第3条第1款,证据材料第5页）
```

**格式2: 法律文书引用** (Legal Document Citation)
```
[事实陈述]（见[法院名称][案号]民事判决书第X页）

示例:
"法院认定欠款本金100万元"（见慈溪市人民法院(2023)浙0282民初1234号民事判决书第8页）
```

**格式3: 单据凭证引用** (Voucher Citation)
```
[事实陈述]（见[单据类型],证据材料第X页）

示例:
"2024年1月15日转账100万元"（见银行转账凭证,证据材料第12页）
"开具增值税专用发票50万元"（见发票号23420240101234,证据材料第15页）
```

**格式4: 多处证据交叉引用** (Cross-Reference)
```
[事实陈述]（见[证据A]第X页、[证据B]第X页）

示例:
"放款金额为100万元"（见《借款合同》第2条,证据材料第3页、银行转账凭证,证据材料第18页）
```

**格式5: 证据缺失标注** (Missing Evidence Notation)
```
[事实项目]: [证据未记载/债权人未提供/无法确定]

示例:
借款期限: [合同未约定]
还款日期: [债权人未提供还款凭证]
利率标准: [证据未明确记载,待补充]
```

---

#### ❌ 禁止的模糊引用 (Prohibited Vague References)

**禁止1: 无具体来源**
```
❌ 错误: "根据证据材料,借款金额为100万元"
✅ 正确: "借款金额为100万元（见《借款合同》第2条,证据材料第5页）"

理由: "证据材料"太宽泛,无法定位具体页码
```

**禁止2: 无页码定位**
```
❌ 错误: "根据判决书,债务人应付利息"
✅ 正确: "债务人应付利息（见(2023)浙0282民初1234号判决书第7-8页）"

理由: 缺少页码,复核时无法快速定位
```

**禁止3: 来源混淆**
```
❌ 错误: "根据债权人说明,双方约定利率为6%"
✅ 正确:
- 如合同有约定: "约定利率为6%（见《借款合同》第4条,证据材料第6页）"
- 如仅债权人主张: "利率约定: [合同未记载,债权人申报为6%但未提供依据]"

理由: "债权人说明"是单方陈述,不能作为事实依据
```

**禁止4: 间接引用**
```
❌ 错误: "判决书提到合同约定了利率,因此利率有约定"
✅ 正确:
- 如判决明确载明: "约定利率为5.8%（见判决书第5页引述合同第4条）"
- 如判决未明确: "利率: [判决书未明确记载具体数值,债权人未提供合同原件]"

理由: 不能用判决的"提到"推测合同内容
```

**禁止5: 证据目录引用**
```
❌ 错误: "根据证据清单显示,债权人提供了15份证据"
✅ 正确: 直接引用具体证据内容,不引用清单

理由: 证据清单是索引,不是证据本身
```

**禁止6: 多重推理链**
```
❌ 错误: "判决支持利息请求 → 说明有利率约定 → 利率应该合理 → 我们认为利率为LPR"
✅ 正确: "利率: [判决书未明确记载利率标准]"

理由: 禁止基于判决结果反推事实细节
```

**禁止7: 常识填充**
```
❌ 错误: "建设工程合同一般包含质保条款,因此本案应有质保金"
✅ 正确: "质保金: [合同未约定]"

理由: 不能用"一般情况"代替具体证据
```

#### 特殊证据类型的引用要求

**类型A: 判决书/裁定书/仲裁裁决**
- ✅ 必须引用: 案号 + 页码
- ✅ 必须区分: 判决"认定的事实"部分 vs "本院认为"部分
- ❌ 禁止引用: 判决书的"原告诉称"/"被告辩称"作为事实（这些是主张,不是认定）

**示例**:
```
✅ 正确: "法院认定放款日期为2024年1月10日（见判决书第3页'经审理查明'部分）"
❌ 错误: "债权人主张放款日期为2024年1月10日（见判决书'原告诉称'部分）"→应标注为债权人主张,非法院认定事实
```

**类型B: 合同及补充协议**
- ✅ 必须引用: 合同名称 + 条款编号 + 页码
- ✅ 修改关系: 如补充协议修改原合同,必须说明"原约定X,后修改为Y"
- ❌ 禁止: 只引用补充协议而不说明原合同

**示例**:
```
✅ 正确: "利率原约定为6%（见《借款合同》第4条,第2页）,后双方于2024年6月签订补充协议,利率调整为5%（见《补充协议》第1条,第15页）"
❌ 错误: "利率为5%（见补充协议）" → 未说明修改前后关系
```

**类型C: 银行流水/转账凭证**
- ✅ 必须包含: 转账日期 + 金额 + 转出账户后4位 + 转入账户后4位 + 页码
- ✅ 必须说明: 摘要/用途栏内容（如"借款"、"货款"、"往来款"）
- ❌ 禁止: 省略摘要信息（摘要是判断款项性质的关键）

**示例**:
```
✅ 正确: "2024年1月10日,从尾号1234账户转账100万元至尾号5678账户,摘要为'借款'（见银行转账凭证,第8页）"
❌ 错误: "2024年1月10日转账100万元" → 缺少账户信息和摘要
```

**类型D: 发票**
- ✅ 必须包含: 发票类型 + 发票号码 + 开票日期 + 含税金额 + 页码
- ✅ 必须说明: 发票备注栏内容（如有）
- ❌ 禁止: 用发票证明"已付款"（发票只证明开票,不证明付款）

**示例**:
```
✅ 正确: "开具增值税专用发票,号码23420240101234,含税金额50万元,开票日期2024年3月1日,备注栏注明'工程款第一期'（见发票,第10页）"
注意: 不可写"支付工程款50万" → 发票≠付款凭证
```

**类型E: 送货单/验收单**
- ✅ 必须包含: 单据日期 + 货物名称/数量 + 签收人姓名 + 页码
- ✅ 必须说明: 是否有签收（签收≠付款）
- ❌ 禁止: 用送货单证明"已验收"（送货≠验收,需验收单）

**示例**:
```
✅ 正确: "2024年2月15日送货,货物为钢材50吨,送货单有'张三'签收（见送货单,第12页）。注: 无验收单,是否验收合格待查。"
❌ 错误: "2024年2月15日交付钢材50吨并验收合格" → 送货单不能证明验收合格
```

**类型F: 对账单/欠款确认函**
- ✅ 必须包含: 确认日期 + 确认金额 + 签字/盖章情况 + 页码
- ✅ 必须区分: 双方盖章 vs 单方盖章（效力不同）
- ❌ 禁止: 省略签章情况

**示例**:
```
✅ 正确: "《对账单》显示截至2024年12月31日欠款100万元,债务人盖章确认（见对账单,第20页）"
✅ 正确: "《对账单》显示欠款100万元,仅债权人单方盖章,债务人未签章（见对账单,第20页）"
❌ 错误: "对账单确认欠款100万" → 未说明是否债务人确认
```

#### 📋 引用自查清单 (Citation Self-Check)

**在完成每一段事实陈述后,必须自查以下问题:**

**检查1: 来源明确性**
```
□ 每个事实都标注了具体证据来源?
□ 证据名称完整（合同全称、判决案号、单据类型）?
□ 页码准确标注?
```

**检查2: 引用具体性**
```
□ 是否使用了"根据证据材料"等模糊表述?（❌ 禁止）
□ 是否使用了"债权人说明""债权人提供的材料显示"?（❌ 禁止,应指明具体材料）
□ 是否引用了证据清单/目录?（❌ 禁止）
```

**检查3: 证据类型匹配**
```
□ 用发票证明"开票事实" ✅
□ 用发票证明"付款事实" ❌ （发票不能证明付款）
□ 用送货单证明"送货事实" ✅
□ 用送货单证明"验收合格" ❌ （送货单不能证明验收）
□ 用对账单证明"对账结果" → 必须说明是否双方确认
```

**检查4: 缺失标注**
```
□ 证据未记载的事项,是否用"[证据未记载]"明确标注?
□ 债权人未提供的材料,是否用"[债权人未提供]"标注?
□ 证据模糊的事项,是否用"[证据未明确,待补充]"标注?
□ 是否试图用"常识""惯例"填补证据空白?（❌ 禁止）
```

**检查5: 推理边界**
```
□ 是否基于判决结果反推事实细节?（❌ 禁止）
□ 是否基于"一般情况"假设未明事项?（❌ 禁止）
□ 是否跨证据类比填补?（❌ 禁止）
□ 是否用间接证据推测直接事实?（❌ 禁止）
```

---

#### 违规后果

**如报告中存在以下情况,将被退回重做:**
1. 使用"根据证据材料"等模糊引用（来源不明）
2. 缺少页码定位（无法复核）
3. 混淆证据类型与证明对象（如用发票证明付款）
4. 引用证据清单/目录作为事实依据（清单非证据）
5. 用判决的"原告诉称"作为事实（主张非认定）
6. 省略签章情况（对账单/确认函效力不明）
7. 使用"债权人说明"作为事实依据（单方主张非证据）

**核心原则**: 每个事实都必须能够在30秒内定位到具体证据页码并复核。如无法快速定位,说明引用不合格。

## Part 2: Classified Fact-Finding Standards

### Contract Type 1: Sales/Supply Contracts

**Standard Output Format** (9 Core Clauses Required):

```
根据[债权人全称]与[债务人全称]于[签订日期]签订的《[合同名称]》（编号：[编号]），债权人向债务人供应[货物简述]，合同总价款为人民币[金额]元。核心约定如下：

1. 标的物：[产品/项目名称]（见合同第[X]条）

2. 合同价款：含税总金额为人民币[金额]元（见合同第[X]条第[X]款）

3. 履行期限：[具体起止日期或履行期限要求]（见合同第[X]条）

4. 交付与验收：[交付地点、验收标准、签收要求]（见合同第[X]条）

5. 付款方式：[完整摘录付款条件关键原文，包括付款条件、付款期限、前置条件等，如"货到验收合格后7日内付款"、"收到发票后30日内付款"等]（见合同第[X]条第[X]款）

6. 违约责任：[逾期付款违约金率、逾期交货违约责任等]（见合同第[X]条）

7. 发票要求：[开票条件、开票时间要求、含税总金额等]（见合同第[X]条）

8. 争议解决：[仲裁/诉讼管辖约定]（见合同第[X]条）

9. 签收条款：[必须明确指定签收人姓名/职务，如"指定张三为唯一签收人"等]（见合同第[X]条）
```

**⚠️ Special Attention**:
- **Payment Terms Clause**: Must include ALL preconditions (e.g., "after delivery and acceptance", "after receiving invoice")
- **Recipient Clause**: Must specify WHO can sign (name or position)
- **Invoice Clause**: Must include timing, amount, conditions

### Contract Type 2: Service/Construction Contracts

**Standard Output Format** (Similar 9-clause structure):

```
根据[债权人全称]（承包人/服务方）与[债务人全称]（发包人/委托方）于[签订日期]签订的《[合同名称]》（编号：[编号]），债权人承接[工程项目/服务内容]，合同总价款为人民币[金额]元。核心约定如下：

1. 服务内容：[具体服务项目/工程内容]（见合同第[X]条）

2. 合同价款：暂定/确定含税总金额为人民币[金额]元（见合同第[X]条第[X]款）

3. 服务期限：自[起始日期]至[结束日期]（见合同第[X]条）

4. 验收确认：[验收文件类型、签字人员要求、验收标准]（见合同第[X]条）

5. 结算确认：[结算文件要求、确认流程]（见合同第[X]条第[X]款）

6. 付款方式：[完整摘录付款条件关键原文，包括各阶段付款比例、付款条件、付款期限等]（见合同第[X]条第[X]款）

7. 违约责任：[逾期完工违约金、质量违约责任等]（见合同第[X]条）

8. 质保约定：[质保期限、质保金比例]（见合同第[X]条）

9. 其他关键约定：[变更、索赔、发票要求等]（见合同第[X]条）
```

### Contract Type 3: Loan Contracts

**Standard Output Format**:

```
根据[债权人全称]与[债务人全称]于[签订日期]签订的《[合同名称]》（编号：[编号]），债权人向债务人提供借款合计人民币[借款本金]元，约定利率为[利率标准]，借款期限自[起算日]至[到期日]。核心约定如下：

1. 借款本金：人民币[金额]元（大写：[金额大写]）（见合同第[X]条）

2. 借款期限：自[起始日期]至[到期日期]，共[期限]（见合同第[X]条）

3. 交付约定：[资金交付方式、指定账户信息]（见合同第[X]条第[X]款）

4. 利息约定：期内年利率为[X%/LPR+X基点]，[固定利率/浮动利率]，[付息方式]（见合同第[X]条第[X]款）

5. 逾期责任：[逾期利率、罚息计算方式]（见合同第[X]条）

6. 还款约定：[一次性还本付息/等额本息/等额本金/其他]，[还款账户]（见合同第[X]条第[X]款）

7. 担保措施：[担保人、担保物等]（见合同第[X]条）

8. 追索费用：[律师费、诉讼费等约定]（见合同第[X]条）
```

### Evidence Type 4: Invoices

**Standard Output Format**:

```
根据[销售方全称]于[开票日期]开具的增值税[专用/普通]发票（发票号码：[号码]），[销售方]向[购买方全称]销售[货物或服务名称]，价税合计人民币[含税总金额]元。发票基本信息如下：

- 发票类型：[增值税专用发票/增值税普通发票/其他]
- 发票代码：[发票代码]
- 发票号码：[发票号码]
- 开票日期：[YYYY年MM月DD日]
- 购买方（付款方）：[单位全称]，纳税人识别号：[号码]
- 销售方（收款方）：[单位全称]，纳税人识别号：[号码]
- 货物或应税劳务、服务名称：[具体内容]
- 金额：[不含税金额]元（仅在发票上明确显示时记录）
- 税额：[税额]元（仅在发票上明确显示时记录）
- 价税合计：[含税总金额]元（大写：[大写金额]）
- 备注栏：[如有备注内容]
```

**Multiple Invoices Handling**:
```
发票开具情况：
```

## Part 3: Evidence Relationship Analysis

### ⚠️ CRITICAL: 禁止推理填空规则 (Anti-Inference Gap-Filling Rule)

**在进行证据关系分析前,必须牢记以下禁止行为:**

证据关系分析的边界:
- ✅ **允许**: 追溯现有证据之间的覆盖、修改、补充关系
- ❌ **禁止**: 推测缺失证据的内容或效力
- ❌ **禁止**: 用"常理""惯例""一般情况"填补证据空白

---

#### 禁止行为清单(严格执行)

**❌ 禁止1: 推测缺失文件内容**
```
错误示例:
"判决书引用了《借款合同》第5条,虽然我们没有合同原件,
但根据判决书描述,合同约定利率应该是6%"

为何错误:
- 判决书引用≠判决书载明
- 没有合同原件=无法确定合同条款
- 不能用判决的"描述"反推合同"内容"

正确做法:
"判决书引用《借款合同》第5条,但债权人未提供合同原件。
判决书未直接载明利率数值,仅表述'按约定利率计算'。
[合同约定利率:无法确定,债权人未提供合同原件]"
```

**❌ 禁止2: 用"常理"填补证据空白**
```
错误示例:
"合同未约定付款方式,根据建筑行业惯例,应为分期付款"

为何错误:
- "行业惯例"不是法律依据,更不是本案证据
- 不能用普遍做法推定具体合同约定
- 具体合同完全可以偏离"惯例"

正确做法:
"合同未约定付款方式。
[付款方式:合同未记载,证据不足以确定]"
```

**❌ 禁止3: 用"一般情况"假设未明事项**
```
错误示例:
"虽然送货单未签收,但货物已交付,
按一般交易习惯应视为验收合格"

为何错误:
- 交付≠验收
- "一般习惯"不能替代证据
- 未签收的原因可能很多(拒收、质量问题等)

正确做法:
"送货单无签收记录。
[是否验收:证据未记载]
备注:债权人主张已交付但缺验收凭证"
```

**❌ 禁止4: 跨证据类比填补**
```
错误示例:
"第1笔借款合同约定利率6%,第2笔合同未写利率,
两笔性质相同,第2笔应该也是6%"

为何错误:
- 每笔债权关系独立
- 不同合同可以有不同约定
- 不能用A合同推测B合同

正确做法:
"第2笔借款合同未约定利率。
[第2笔利率:合同未记载]
备注:第1笔合同约定6%,但不能推定第2笔相同"
```

**❌ 禁止5: 用判决"隐含意思"推理事实**
```
错误示例:
"判决支持了利息请求,虽然未明确利率,
但判决支持说明合同应该有约定,我们可以认为有利率约定"

为何错误:
- 判决支持利息≠判决明确利率
- 判决支持可能基于多种理由(法定利率、推定等)
- 不能用判决结果反推合同条款

正确做法:
"判决支持利息请求但未载明利率依据。
[利率约定:判决书未明确记载]
备注:判决支持利息,但未明确利率标准"
```

**❌ 禁止6: 填补证据之间的逻辑缺口**
```
错误示例:
"合同约定分3期付款,仅有第1、3期付款凭证,
第2期应该也已支付(否则不会支付第3期)"

为何错误:
- 这是逻辑推测,不是证据认定
- 第3期支付的原因可能很多(催收、法律压力等)
- 缺失证据=事实不清

正确做法:
"合同约定分3期付款。
已支付:第1期(凭证X),第3期(凭证Y)。
第2期:债权人未提供付款凭证。
[第2期是否支付:证据不足以确定]"
```

---

#### ✅ 允许的证据分析行为

**允许1: 追溯证据修改链**
```
✅ 正确:
"合同A被补充协议B修改第3条,
补充协议B又被补充协议C修改第3条,
根据证据修改链,第3条最终以补充协议C为准"

理由:这是追溯现有证据,不是推测缺失内容
```

**允许2: 识别证据覆盖关系**
```
✅ 正确:
"判决书确定本金为100万元,
虽然合同约定本金120万元,
根据证据效力原则,判决书优于合同,
本金以判决书确定的100万元为准"

理由:这是应用证据效力原则,不是推测
```

**允许3: 记录证据冲突**
```
✅ 正确:
"证据X(合同)记载金额100万元,
证据Y(对账单)记载金额80万元,
两者矛盾。
[实际金额:证据冲突,待进一步查明]"

理由:如实记录矛盾,不擅自选择
```

**允许4: 标注证据缺失**
```
✅ 正确:
"需要收货凭证证明交付事实,
但债权人未提供收货凭证。
[收货事实:证据不足]"

理由:明确标注缺失,不填补空白
```

**允许5: 引用法律规定进行分类**
```
✅ 正确:
"根据《民法典》第787条,本合同构成保证合同"

理由:这是法律分类,不是事实推测
(注意:仍需证据支持构成要件事实)
```

#### 🎯 判断标准:这是分析还是推测?

**自检问题**:
1. **我的结论是否能在证据原文中找到对应表述?**
   - 能找到 → 可能是证据分析 ✅
   - 找不到 → 可能是推测编造 ❌

2. **我是否使用了"应该""可能""根据常理""一般来说"?**
   - 是 → 这是推理词汇,警惕! ⚠️
   - 否 → 继续检查

3. **我是否在用A证据的内容去"推测"B证据的缺失部分?**
   - 是 → 这是禁止的填补 ❌
   - 否 → 可能合法

4. **去掉这个推测,结论会有证据空白吗?**
   - 是 → 说明这是在填补空白,应停止 ❌
   - 否 → 可能是基于现有证据的分析 ✅

5. **我能用"根据[具体证据编号X,第Y页]"来支持这个结论吗?**
   - 能 → 可能是证据分析 ✅
   - 不能 → 可能是推测 ❌

**缺失标记格式**:

- `[XX事项:证据不足]` - 证据不足以确定

- ✗ 后续的法律分析和金额确认全部无效
- ✗ 可能误导客户决策,造成严重法律后果

### Evidence Hierarchy (Highest to Lowest)

1. **Effective Legal Documents** (最高效力)
   - Court judgments, mediation agreements, arbitration awards
   - Override all other evidence

2. **Bilateral Confirmation Documents**
   - Debt confirmation letters, reconciliation statements, settlement documents
   - Override performance evidence and original contracts (for amounts confirmed)

3. **Supplementary/Amendment Agreements**
   - Modify specific terms of original contract
   - Only modified terms supersede original

4. **Original Contracts/Agreements**
   - Foundation of relationship
   - Terms apply unless modified by higher-hierarchy evidence

5. **Unilateral Performance Evidence**
   - Invoices, delivery slips, bank transfers
   - Prove facts but don't create new terms

6. **Unilateral Declaration Materials**
   - Lowest hierarchy
   - Cannot be used for fact-finding

### Coverage Relationships

**What "Coverage" Means**:
- Later document with higher hierarchy overrides earlier/lower documents
- Only applies to SAME subject matter

**Examples**:

1. **Settlement Document Covers Contract**:
   - Original contract: Payment due 2023-06-01
   - Settlement document (2023-08-15): Confirms 100万 owed, new payment plan
   - **Result**: Use settlement document for amount and payment terms, contract provides background

2. **Judgment Covers All**:
   - Original contract: Interest at 6% per year
   - Judgment: Interest calculated at LPR
   - **Result**: Use judgment's interest standard, ignore contract's

3. **Confirmation Letter Covers履约细节**:
   - Multiple invoices and payments over time
   - Confirmation letter: As of 2023-12-31, total owed is 50万
   - **Result**: Use 50万 as confirmed amount, earlier invoices/payments are historical

### Reference Relationships

**What "Reference" Means**:
- Higher document refers back to lower document for specific terms

**Common Scenario**: Judgment References Original Contract

Example:
```
Judgment says: "债务人应支付本金100万元，利息按原借款合同约定计算"

Analysis Required:
1. Principal amount: Per judgment (100万)
2. Interest standard: Must trace back to original contract
3. Find original contract clause about interest (e.g., Article 5: "年利率6%")
4. Apply: Principal 100万 from judgment + Interest rate 6% from contract
```

**How to Handle**:
1. Identify the reference phrase (e.g., "按原合同约定", "依据借款协议")
2. Locate the referenced document
3. Find the specific clause referenced
4. Quote both judgment AND original clause
5. Explain mixed application

### Multiple-Layer Modifications

**Scenario**: Original Contract → Supplement 1 → Supplement 2 → Confirmation Letter

**How to Analyze**:
1. Identify what each layer modified
2. Trace the modification chain
3. Determine final applicable term for each element

**Example**:
```
Original Contract (2023-01-01):
- Payment: 30 days after delivery
- Interest: 6% per year

Supplement 1 (2023-03-15):
- Payment: Changed to 60 days after delivery
- (Interest not mentioned → original still applies)

Supplement 2 (2023-06-20):
- Interest: Changed to LPR + 100 basis points
- (Payment not mentioned → Supplement 1 still applies)

Confirmation Letter (2023-09-10):
- Confirms total owed amount, sets new payment plan
- (Interest calculation not mentioned → Supplement 2 still applies)

Final Applicable Terms:
- Amount: Per Confirmation Letter
- Payment: Per Confirmation Letter's new plan
- Interest calculation for past period: Per Supplement 2 (LPR+100bp)
```

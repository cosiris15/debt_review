# 债权分析报告模板与标准

## 📋 文档说明
本文档为债权分析员专用参考文件，包含：
- 输出格式标准
- 报告模板
- 完整示例
- 计算过程表格文件说明

**⚠️ 重要原则：**
- 基于事实核查报告进行深入分析
- 精确计算各项金额，生成计算过程表格文件
- 每笔债权独立分析，独立生成报告和计算文件

## ⚠️ 模板使用强制规则 (MANDATORY)

**在使用本模板生成报告前,必须阅读并执行以下规则:**

### 占位符替换协议

**规则1: 识别占位符**
- 格式: `[占位符名称]` - 必须替换
- 示例: `[金额]`、`[利率]`、`[日期]`、`[计算公式]`等

**规则2: 定位实际内容**
- 来源: 事实核查报告 / 证据材料 / Calculator工具输出
- 要求: 内容必须有明确依据

**规则3: 替换决策**
```
✅ 有证据/申报依据 → 照抄实际数据
✅ 有Calculator输出 → 照抄计算结果
❌ 证据不足 → 使用标准缺失标记:
   - [证据未明确] - 证据中无法确定
   - [参数缺失,无法计算] - Calculator无法执行
   - [债权人未申报,不予确认] - 就无原则
   - [待补充] - 需要补充材料
```

### ❌ 严禁的占位符填充行为

**禁止1: 推测计算参数**
```
❌ [利率] → "4.35%" (假设为当前LPR)
❌ [LPR期限] → "1年期" (推测应用期限)
✅ [利率] → 照抄合同约定,或标记[合同未约定,无法计算]
```

**禁止2: 手工计算填充**
```
❌ [利息金额] → "50,000元" (心算或Excel手工算)
❌ [计算公式] → "100万 × 4.35% × 1年 = 4.35万" (手工计算)
✅ [利息金额] → 必须使用Calculator工具,填入工具输出结果
```

**禁止3: 估算填充**
```
❌ [金额] → "约100万元" (估算)
❌ [期限] → "大概2年" (估计)
✅ [金额] → 照抄精确数值,或标记[证据未明确]
```

**禁止4: 添加未申报项**
```
❌ [违约金] → "10万元" (虽然合同有约定,但债权人未申报)
❌ [利息] → 主动计算(虽然合同有利率,但债权人未申报利息)
✅ [违约金] → [债权人未申报,不予确认](就无原则)
```

**禁止5: 用就低原则猜参数**
```
❌ [LPR期限] → "选1年期(因为算出来更低)"
   错误:就低原则用于比较金额,不用于选择参数
✅ [LPR期限] → [合同未明确,证据不足,无法计算]
```

### 🧮 Calculator工具强制使用规则

**规则: 所有利息/违约金计算必须使用Calculator**

**✅ 正确流程**:
1. 确定计算参数(本金、日期、利率等)
2. 执行Calculator命令:
   ```bash
   python universal_debt_calculator_cli.py [mode] \
     --principal XXX \
     --start-date YYYY-MM-DD \
     --end-date YYYY-MM-DD \
     [其他参数]
   ```
3. 照抄Calculator输出结果
4. 保存Calculator生成的Excel/CSV文件到`计算文件/`
5. 在报告中完整记录Calculator命令

**❌ 禁止行为**:
- ❌ 心算: "100万 × 4% = 4万,差不多"
- ❌ Excel手算: 自己建Excel公式计算
- ❌ 估算: "按天算太复杂,按年估计就行"
- ❌ 省略命令记录: 只写结果,不记录Calculator命令

**⚠️ 缺失计算文件的处理**:
如果某项确实无需计算(如:债权人未申报利息):
```
必须在`计算文件/`目录创建说明文件:
文件名: {债权人名称}_无计算项说明.txt
内容: "本债权无需计算项目,原因:[债权人未申报利息/仅为固定本金等]"
```

### 交付前检查 (MANDATORY)

**步骤1: 搜索未替换占位符**
```bash
grep -n "\[.*\]" your_analysis_report.md
```

**步骤2: 验证计算文件**
```bash
ls -lh 计算文件/
# 必须存在:
# - *.xlsx 或 *.csv (如有计算项)
# - *_无计算项说明.txt (如无计算项)
# 不应存在:
# - 空目录
# - 仅有说明文档,但报告中包含计算结果
```

**步骤3: 验证Calculator命令记录**
- [ ] 每个利息/违约金计算都记录了完整Calculator命令
- [ ] 命令包含全部参数(--principal, --start-date等)
- [ ] 命令格式正确,可以复现执行

**步骤4: 验证就无/就低原则**
- [ ] 债权人未申报的项,是否标记为"不予确认"
- [ ] 计算结果>申报金额的项,是否确认为申报金额
- [ ] 没有主动添加未申报项

### 占位符使用示例

**示例1: Calculator计算填充**
```
模板: 利息:[利息金额],计算过程:[Calculator命令]
事实: 合同约定年利率6%,本金100万,期间2年
报告:
  利息:12万元
  计算过程:
  python universal_debt_calculator_cli.py simple \
    --principal 1000000 \
    --start-date 2022-01-01 \
    --end-date 2024-01-01 \
    --rate 6.0
  详见计算文件:XX公司_本金利息计算.xlsx ✅
```

**示例2: 参数缺失停止**
```
模板: 利息:[利息金额]
事实: 合同约定"按LPR计息",未明确1年期或5年期
报告: 利息:[LPR期限未明确,无法计算]
      建议:要求债权人补充说明LPR期限 ✅
```

**示例3: 未申报项处理**
```
模板: 违约金:[违约金金额]
事实: 合同约定违约金,但债权人未申报
报告: 违约金:[债权人未申报,不予确认]
      备注:合同第8条约定违约金条款 ✅
```

**示例4: 就低原则应用**
```
模板: 利息:[确认金额]
事实: 债权人申报50万,Calculator计算80万
报告: 利息:50万元(就低原则,申报<计算)
      计算验证:80万元(详见计算文件)
      确认金额:50万元 ✅
```

### 违反规则的后果

- ❌ 手工计算 → 可能产生数十万甚至上百万误差
- ❌ 推测参数 → 计算结果毫无法律效力
- ❌ 添加未申报项 → 违反就无原则,擅自扩大债权
- ❌ 缺失计算文件 → 计算过程无法审计验证

**请在使用模板前仔细阅读以上规则,在交付报告前执行检查程序。**

# 上海金桥信息股份有限公司债权分析报告

## 一、债权基础法律关系确认

基于《上海金桥信息股份有限公司事实核查报告》，确认：
- 独立债权关系数量：1个
- 债权类型：法律文书确认类债权
- 具体识别：（2025）沪0120民初947号民事调解书确认的债权
- 法律关系性质：建设工程施工合同纠纷

## 二、金额项目拆解分析

### 2.1 金额项目清单
根据（2025）沪0120民初947号调解结果，被告同意支付原告工程款685,562.00元及案件受理费5,328.00元。

| 序号 | 项目描述 | 申报金额 | 法律性质 | 证据支持情况 | 计算需求 |
|------|----------|----------|----------|--------------|----------|

| 2 | 案件受理费 | 5,328.00元 | 费用 | 有（调解书确认） | 固定 |
| 3 | 迟延履行期间加倍部分债务利息 | 5,843.98元 | 劣后债权 | 需计算验证 | 需计算 |

### 2.2 拆解说明
根据调解书内容，债权构成清晰：
- 工程款本金685,562.00元为主债权
- 案件受理费5,328.00元为诉讼费用
- 迟延履行期间的加倍部分债务利息需根据分期付款安排计算

### 4.1 计算命令记录


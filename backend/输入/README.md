# 债权申报材料存放指南

## 📁 目录说明

本目录用于存放破产债权审查的**原始申报材料**，按批次组织。

### 目录结构

```
输入/
├── 第1批债权/          # 第一批债权人申报材料
├── 第2批债权/          # 第二批债权人申报材料
├── 第3批债权/          # 第三批债权人申报材料
└── ...                # 后续批次
```

## 📝 文件命名规范

### 标准格式（两种均可）

**格式1：点号分隔**
```
编号.债权人名称.md
```
示例：`115.慈溪市东航建筑起重机械安装队.md`

**格式2：横线分隔**
```
编号-债权人名称.md
```
示例：`35-天津驹丰供应链管理有限公司.md`

### 命名要求

1. **编号**：债权人编号，通常为数字
2. **债权人名称**：完整的债权人名称（个人或企业）
3. **文件格式**：必须为 `.md` 格式（Markdown）

## 📋 材料组织原则

### 1. 按批次分类
- 每个批次的材料放入对应的 `第X批债权/` 目录
- 不同批次的材料**严格分开**，不得混放

### 2. 一债权人一文件
- 每个债权人的申报材料整理为**一个**独立的 `.md` 文件
- 文件内容应包含：
  - 债权申报登记表
  - 债权申报书
  - 身份证明文件
  - 证据材料（合同、发票、法律文书等）

### 3. 保持原始性
- 输入材料应保持原始状态
- 处理结果将自动输出到 `/输出/` 目录
- **不要在输入目录中修改原始文件**

## 🔄 工作流程

### 材料流转

```
1. 准备阶段
   └─> 将债权申报材料整理为 .md 格式
   └─> 按批次放入对应的 第X批债权/ 目录

2. 初始化阶段
   └─> 运行: python 债权处理工作流控制器.py <批次号> <编号> <债权人名称>
   └─> 系统自动创建输出目录结构

3. 处理阶段
   └─> 系统从 输入/ 目录读取原始材料
   └─> 逐步处理：事实核查 → 债权分析 → 报告整理
   └─> 结果自动输出到 /输出/ 目录

4. 完成阶段
   └─> 查看 /输出/第X批债权/[编号]-[债权人名称]/ 目录
   └─> 获取最终审查报告和计算文件
```

## ⚠️ 注意事项

### 重要提醒

1. **输入材料只读**
   - 本目录的材料仅用于读取
   - 处理过程不会修改输入文件
   - 所有输出都在 `/输出/` 目录

2. **文件格式要求**
   - 必须为 `.md` 格式
   - 编码必须为 UTF-8
   - 确保文件可读性

3. **隐私保护**
   - 输入材料可能包含敏感信息
   - 建议在版本控制中排除本目录
   - 注意数据安全和隐私保护

4. **批次管理**
   - 新增批次时，创建 `第X批债权/` 目录
   - 更新 `project_config.ini` 中的批次列表
   - 保持批次编号连续性

## 🆘 常见问题

### Q1: 如何添加新批次？
**A:**
1. 在 `输入/` 目录下创建 `第X批债权/` 文件夹
2. 更新 `project_config.ini` 中的 `债权材料目录` 配置
3. 将新批次的材料放入新建的目录

### Q2: 文件名称格式错误会怎样？
**A:**
- 系统可能无法正确识别债权人信息
- 建议严格按照命名规范组织文件
- 出现问题时检查文件名格式

### Q3: 可以直接在这里修改文件吗？
**A:**
- 不建议直接修改输入材料
- 如需修正，应在源头修正后重新放入
- 处理结果的修改应在 `/输出/` 目录进行

### Q4: 输入材料可以删除吗？
**A:**
- 处理完成后，建议保留原始材料作为档案
- 如需清理，确保已完成所有处理并备份输出结果
- 删除前确认不再需要重新处理

## 🎯 快速决策：是否使用并行处理？

### 决策树

```
有多少个债权人需要处理？
├─ 1个 → 使用串行处理（不必并行）
├─ 2-5个 → ✅ 推荐并行处理（最佳性价比）
├─ 6-15个 → ✅ 推荐分批并行（每批5-8个）
└─ 15+个 → ✅ 推荐多批并行（每批5-8个）

您熟悉系统吗？
├─ 第一次使用 → 先用串行处理，熟悉后再尝试并行
├─ 已经熟练 → ✅ 可以使用并行处理
└─ 正在调试问题 → 使用串行处理，便于定位问题

材料完整吗？
├─ 所有材料齐全 → ✅ 可以并行
├─ 部分材料缺失 → 不推荐并行（失败率高）
└─ 材料质量差 → 不推荐并行

有充足时间准备吗？
├─ 可以花10-15分钟准备 → ✅ 可以并行
└─ 赶时间，立即开始 → 使用串行（准备时间短）
```

### 并行 vs. 串行对比

| 维度 | 串行处理 | 并行处理 |
|------|---------|---------|
| **效率（5个债权人）** | 80分钟 | 18分钟 |
| **准备时间** | 5分钟 | 10-15分钟 |
| **操作复杂度** | 简单 | 中等 |
| **失败处理** | 简单 | 需要识别失败债权人 |
| **适用场景** | 1-2个债权人 | 3+个债权人 |
| **学习曲线** | 平缓 | 需要理解prompt模板 |

## 🔧 常见问题与排查

### 问题1: 某个Agent执行失败

**症状**: 5个Task中有1-2个失败

**排查**:
```bash
# 查看失败Agent的错误信息
# (系统应该会返回具体的错误)

# 常见错误类型:
# - 配置验证失败: 检查 .processing_config.json
# - 文件不存在: 检查输入材料路径
# - 路径错误: 检查prompt中的路径是否正确
```

**解决方案**:
1. 分析错误原因（配置、路径、材料）
2. 修复问题
3. **仅重新处理失败的债权人**（单独调用，不需要重新处理成功的）
4. 重新执行质量检查

### 问题2: 上下文污染（债权人信息混淆）

**症状**: 债权人A的报告中出现债权人B的信息

**排查**:
```bash
# 检查报告中的债权人名称
grep "债权人名称" 输出/第1批债权/115-*/工作底稿/*事实核查报告.md

# 在债权人A的报告中搜索债权人B的名称
grep "徐小勇" 输出/第1批债权/115-*/**/*.md
```

**解决方案**:
1. **立即停止**使用并行处理
2. 删除有问题的报告: `rm -rf 输出/第1批债权/115-*`
3. 重新初始化: `python 债权处理工作流控制器.py 1 115 慈溪市东航建筑起重机械安装队`
4. 使用**串行模式**重新处理该债权人（确保无污染）
5. 检查prompt是否完全自包含
6. 如问题持续，报告系统bug

### 问题3: 日期不一致

**症状**: 不同报告中的破产日期不一致

**排查**:
```bash
# 检查所有报告的破产日期
grep -r "破产受理日期" 输出/第1批债权/115-*/**/*.md

# 检查配置文件
cat 输出/第1批债权/115-*/.processing_config.json | grep bankruptcy_date
```

**解决方案**:
1. 确认 `project_config.ini` 中的日期正确
2. 重新初始化所有债权人环境
3. 重新生成prompt（确保日期参数正确）
4. 重新并行处理

### 问题4: 计算文件缺失

**症状**: 阶段2完成后，某些债权人没有计算文件

**排查**:
```bash
# 检查计算文件目录
ls 输出/第1批债权/115-*/计算文件/

# 检查分析报告中是否提到"无需计算"
grep "无需计算\|无计算项" 输出/第1批债权/115-*/工作底稿/*债权分析报告.md
```

**解决方案**:
- 如报告明确说明"无需计算" → 正常，应该有"无计算项说明.txt"
- 如需要计算但文件缺失 → 重新处理该债权人的阶段2

## 💡 最佳实践

### 1. 批量大小选择

**推荐配置**:
```
新手: 2-3个债权人（学习并行流程）
熟练: 5-8个债权人（最佳效率）
大批量: 分批处理，每批5-8个
```

**避免**:
- 一次并行 >10 个债权人（系统负载大，错误难追踪）
- 只并行1个债权人（无效率提升）

### 2. Prompt准备技巧

**使用编辑器的查找替换功能**:
1. 复制模板
2. 替换第一个债权人的所有占位符
3. 复制4份
4. 对每份进行债权人特定的替换

**检查清单**:
```
□ 所有绝对路径正确
□ 债权人编号、名称一致
□ 破产日期正确
```

**关键检查**（必须做）:

## 📊 性能优化建议

### 优化点1: 提前准备所有Prompt

**时间节省**: 5-10分钟

**方法**:
- 在阶段1开始前，准备好所有三个阶段的prompt模板
- 在等待某个阶段完成时，准备下一个阶段的prompt
- 减少阶段间的准备时间

### 优化点2: 并行初始化脚本

**创建初始化脚本**:
```bash
#!/bin/bash
# init_batch.sh

BATCH=1
CREDITORS=(
  "115:慈溪市东航建筑起重机械安装队"
  "118:徐小勇"
  "124:上海合典建材有限公司"
  "134:张之华"
  "140:浙江龙达实业股份有限公司"
)

for creditor in "${CREDITORS[@]}"; do
  IFS=':' read -r num name <<< "$creditor"
  echo "初始化债权人 $num: $name"
  python 债权处理工作流控制器.py $BATCH $num "$name"
done

echo "所有债权人初始化完成"
```

**使用**:
```bash
chmod +x init_batch.sh
./init_batch.sh
```

### 优化点3: 自动化质量检查脚本

**创建检查脚本**:
```bash
#!/bin/bash
# check_stage1.sh

BATCH=1

echo "=== 阶段1质量检查 ==="

echo "1. 文件数量检查"
count=$(find 输出/第${BATCH}批债权 -name "*事实核查报告.md" | wc -l)
echo "事实核查报告数量: $count"

echo "3. 独立性抽查"
# 抽查前两个债权人
echo "检查债权人115报告中是否包含债权人118的名称..."
grep -i "徐小勇" 输出/第${BATCH}批债权/115-*/**/*.md && echo "❌ 发现污染！" || echo "✅ 独立性良好"

echo "=== 检查完成 ==="
```

# 报告整理员工作标准

## 📋 适用范围
本标准适用于报告整理员将事实核查报告和债权分析报告整合成最终审查意见表的工作流程，确保输出的专业性、一致性和标准化。

## 🎯 工作目标
1. **内容整合**：将两份技术报告整合为标准化的审查意见表
2. **格式标准化**：应用客户模板确保输出符合交付要求
3. **文件规范化**：统一文件命名和组织结构

## ⚠️ 工作前置检查（强制执行）

### 破产受理日期核对
**⚠️ 此步骤为强制性检查，开始报告整合工作前必须执行**

1. **读取配置**：从当前债权人目录的 `.processing_config.json` 文件获取破产受理日期
2. **多重交叉验证**：
   - 与事实核查报告中记录的日期对比
   - 与债权分析报告中记录的日期对比
   - 确保三方数据完全一致
3. **记录确认**：在最终报告开头明确记录使用的破产受理日期
4. **异常处理**：如发现任何不一致，立即停止工作并详细报告差异

**示例输出格式**：
```
✅ 破产受理日期核对完成
- 破产受理日期：2023-05-12
- 停止计息日期：2023-05-11
- 配置文件日期：2023-05-12 ✓
- 事实核查报告日期：2023-05-12 ✓
- 债权分析报告日期：2023-05-12 ✓
- 多重验证结果：一致
```

**重要性说明**：报告整理员作为最后一个环节，有责任确保最终交付给客户的报告中所有日期信息准确无误。任何日期错误都将直接影响客户决策的准确性。

## 📊 报告整合标准

### 一、内容提取原则

#### 1.1 内容来源要求
- **唯一性原则**：所有实质内容必须来源于《事实核查报告》和《债权分析报告》
- **完整性原则**：不遗漏关键审查发现和结论
- **准确性原则**：不修改原始数据和分析结论

#### 1.2 内容映射方法
- **映射规则获取**：从审查意见表模板中读取具体的内容映射表
- **灵活匹配**：根据模板定义的章节结构提取对应内容
- **追溯原则**：保持内容来源的可追溯性

### 二、内容重组规则

#### 2.1 信息去重
- 识别两份报告中的重复信息
- 保留最详细或最终版本
- 避免同一信息多次出现

#### 2.2 逻辑重构
- 按客户阅读习惯重新组织内容顺序
- 从结论到过程的展示逻辑
- 突出关键发现和风险提示

#### 2.3 专业术语处理
- 保留必要的法律术语
- 对复杂概念提供简要说明
- 使用客户易懂的表述方式

#### 2.4 写作原则
- **遵循模板风格**：按照客户模板定义的写作风格要求
- **保持一致性**：全文风格保持统一
- **专业表述**：使用规范的法律和金融术语

## 📝 模板应用规则

### 一、模板读取
1. 从`审查意见表模板.md`读取当前客户的模板
2. 识别模板的必填章节和可选章节
3. 理解模板的格式要求和样式规范

### 二、内容填充
1. **必填章节**：确保所有必填章节都有对应内容
2. **可选章节**：根据实际情况决定是否包含
3. **缺失处理**：当模板要求的内容在源报告中不存在时，明确标注

### 三、格式应用
1. **遵循模板格式**：按照客户模板中定义的具体格式要求
2. **标题层级**：严格遵循模板的标题层级结构
3. **保持规范性**：确保格式的一致性和专业性

## 📂 文件命名规范

### 一、主报告命名
```
[项目代码]_[债权人名称]_债权审查报告_[YYYYMMDD].md
```

示例：`GY2025_上海金桥信息_债权审查报告_20250905.md`

### 二、附件命名
```
[项目代码]_[债权人名称]_[文件类型]_[序号]_[YYYYMMDD].[扩展名]
```

文件类型包括：
- 计算明细
- 工作底稿
- 证据材料
- 其他附件

### 三、命名要求
- 使用下划线分隔各部分
- 日期格式统一为YYYYMMDD
- 中文名称保持与申报材料一致
- 避免使用特殊字符

## 🗂️ 目录结构标准

### 标准目录结构
```
输出/第X批债权/[序号]-[债权人名称]/
├── 最终报告/
│   ├── [项目代码]_[债权人名称]_债权审查报告_[日期].md
│   ├── [项目代码]_[债权人名称]_债权审查报告_[日期].pdf (如需要)
│   └── 附件/
│       ├── 计算明细表.xlsx
│       └── 其他支持文档
├── 工作底稿/
│   ├── [债权人名称]事实核查报告.md
│   ├── [债权人名称]债权分析报告.md
│   └── 原始申报材料/
└── 计算文件/
    ├── 利息计算表.xlsx
    ├── 违约金计算表.xlsx
    └── 计算过程说明.txt
```

### 目录说明
- **最终报告**：存放交付客户的所有文件
- **工作底稿**：保留原始技术报告供审计追溯
- **计算文件**：所有计算过程文件集中管理

## ✅ 质量控制要求

### 一、内容完整性检查
- [ ] 所有模板必填章节已完成
- [ ] 关键数据与源报告一致
- [ ] 附件清单与实际文件对应
- [ ] 无遗漏的重要发现

### 二、格式规范性检查
- [ ] 符合模板格式要求
- [ ] 表格显示正常
- [ ] 编号连续正确
- [ ] 无格式错误

### 三、文件规范性检查
- [ ] 文件命名符合规范
- [ ] 目录结构完整
- [ ] 所有引用文件存在
- [ ] 文件可正常打开

### 四、数据一致性检查
- [ ] 申报金额前后一致
- [ ] 确认金额前后一致
- [ ] 计算结果匹配
- [ ] 日期信息准确

## 🔄 工作流程

### 标准处理流程
1. **接收报告**
   - 确认收到完整的事实核查报告
   - 确认收到完整的债权分析报告
   - 确认收到所有计算文件

2. **读取模板**
   - 加载审查意见表模板
   - 理解模板结构和要求

3. **内容提取**
   - 从源报告提取所需内容
   - 建立内容映射关系

4. **报告生成**
   - 按模板填充内容
   - 应用格式规范
   - 生成最终报告

5. **文件整理**
   - 重命名所有文件
   - 组织目录结构
   - 创建文件清单

6. **质量检查**
   - 执行完整性检查
   - 验证数据一致性
   - 确认格式规范

## ⚠️ 注意事项

### 禁止事项
- ❌ 不得修改原始计算结果
- ❌ 不得添加新的分析结论
- ❌ 不得删除重要风险提示
- ❌ 不得改变债权性质认定

### 特殊情况处理
1. **模板缺失**：使用默认结构生成报告
2. **内容冲突**：以债权分析报告为准
3. **格式异常**：保留内容优先，记录格式问题
4. **文件缺失**：明确标注并继续处理

## 📊 报告生成流程

### 处理步骤
1. **读取审查意见表模板** - 从审查意见表模板获取章节结构和映射规则
2. **提取源内容** - 根据映射表从两份技术报告提取内容
3. **内容重组** - 按照模板要求重新组织内容
4. **格式应用** - 应用客户特定的格式和风格要求
5. **质量检查** - 确保输出符合所有标准

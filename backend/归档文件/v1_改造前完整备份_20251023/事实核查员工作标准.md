# 申报梳理要求

## 📋 适用范围
本标准适用于事实核查员对债权申报材料进行系统性梳理，将申报信息结构化整理，为后续债权分析提供清晰的基础信息。本标准包含超长材料分批处理机制，确保能够处理各种规模的债权证据材料。

## 🎯 梳理目标
1. **信息结构化**：将申报材料中的关键信息系统整理
2. **问题识别**：发现申报材料中的疑点和不一致之处

## ⚠️ 工作前置检查（强制执行）

### 破产受理日期核对
**⚠️ 此步骤为强制性检查，开始事实核查工作前必须执行**

1. **读取配置**：从当前债权人目录的 `.processing_config.json` 文件获取破产受理日期
2. **验证有效性**：确认日期格式正确且符合常理
3. **记录确认**：在报告开头明确记录使用的破产受理日期和停止计息日期
4. **异常处理**：如配置文件缺失或日期异常，立即停止工作并报告

**示例输出格式**：
```
✅ 破产受理日期核对完成
- 破产受理日期：2023-05-12
- 停止计息日期：2023-05-11
- 配置文件状态：正常
```

**重要性说明**：破产受理日期是所有利息计算的关键截止点，错误的日期将导致整个债权审查结果失效。

## 📊 申报信息梳理框架

### 一、债权人基本信息梳理

#### 1.1 债权人身份信息
**要求**：准确记录企业债权人的工商信息或个人债权人的身份信息。

#### 1.2 代理信息
**要求**：完整记录代理人信息及代理权限。

### 二、申报债权基本构成梳理

⚠️ **重要提醒：本部分必须严格按照债权人在《债权申报表》《债权申报书》中的实际填写内容完全照抄，不得进行任何主观判断、分类调整或"完善"。即使发现债权人填写有误，也必须原样摘录。**

#### 2.1 债权总额分解
**填写要求：** 严格按照债权人在申报表中的实际填写内容照抄，包括：
- 金额数字必须与原始申报材料完全一致
- 分类名称必须使用债权人申报时的原始表述
- 如债权人未填写某项，则标注"[债权人未填写]"
- 如债权人填写位置与常规不符，必须按实际填写位置记录
- 如债权人将某项费用归类在"其他"而非标准分类，必须按其实际归类记录
- 如债权人漏填或重复填写，如实反映，不得擅自调整

#### 2.2 债权性质分类  
**填写要求：** 严格按照债权人在申报表中勾选或填写的债权性质照抄：
- 债权人勾选了什么性质就记录什么性质
- 债权人没有勾选任何性质的，标注"[债权人未勾选债权性质]"
- 债权人勾选多个性质的，按实际勾选情况全部记录
- 绝对不得根据案件情况主观判断债权性质

#### 2.3 担保情况
**填写要求：** 完全按照债权人在申报材料中的实际表述照抄，如债权人未填写相关内容，标注"[债权人未填写]"。

## 📋 梳理工作核心原则

### 1. 客观性原则
- 严格按照申报材料记录信息，不添加主观判断
- 区分材料中的申报主张与客观证据，申报梳理时不看客观证据
- 发现问题时客观记录，不预设结论

### 2. 完整性原则
- 不遗漏申报材料中的关键信息
- 对重要信息进行交叉核对
- 确保梳理的系统性和全面性

### 3. 准确性原则
- 确保数字、日期、名称等关键信息准确无误
- 对模糊或不清楚的信息注明情况
- 避免因理解偏差导致信息错误

## 📈 质量控制

### 梳理质量标准
- **准确性**：关键信息100%准确
- **完整性**：不遗漏重要信息点
- **逻辑性**：信息整理符合逻辑关系
- **规范性**：输出格式标准统一

### 自检清单
- [ ] 债权人信息是否准确完整
- [ ] 债权金额构成是否严格按申报材料照抄
- [ ] 债权性质是否按债权人实际勾选记录
- [ ] 担保情况是否按债权人实际填写摘录
- [ ] 一致性检查是否到位
- [ ] 问题识别是否客观准确

# 事实查明规范

## 📋 适用范围
本规范适用于事实核查员对债权人提供的证据材料进行系统梳理，查明债权发生的基础事实，确定债权基础法律关系，并按时间顺序整理债权发生的全过程。

## 🎯 查明目标
1. **债权关系识别**：准确识别债权的基础法律关系类型和数量
2. **事实脉络梳理**：按时间顺序还原债权发生、履行、违约的完整过程
3. **证据基础夯实**：确保所有查明情况都有可靠的证据材料支撑
4. **处理能力扩展**：通过分批处理机制处理超长材料

## ⚠️ 核心原则

### 🚨 证据材料区分规则（最重要）
**严格区分证据材料与申报材料：**
- **证据材料**：债权人提供的合同、判决书、调解书、发票等客观证据 → **可作为查明情况依据**
- **申报材料**：债权人提供的申报表、申报书等主观陈述 → **仅作为思考线索，绝对不能作为查明情况的证据使用**
- **核心要求**：查明情况一律以债权人提供的证据材料为准，申报材料只能作为思考线索

## 📦 超长材料分批处理机制

### 触发条件
当证据材料超过系统处理能力时启动分批处理：
- 材料总页数超过100页
- 独立证据项超过50个
- 系统提示材料超出处理窗口

### 分批策略
**基本原则：** 保持关联证据在同一批次，每批控制在30-50个证据项

**分批顺序：**
1. **批次1：核心合同及其补充协议**
   - 主合同及其所有补充协议、变更协议
   - 相关担保合同
   - 与合同直接相关的关键履约证据
   
2. **批次2：履约证据（按时间段分组）**
   - 发票（按时间段分组，如按月或按季度）
   - 送货单、验收单（按时间段分组）
   - 银行转账记录（按时间段分组）
   
3. **批次3：法律文书和确认文件**
   - 判决书、调解书、仲裁裁决
   - 债权确认书、对账单
   - 其他总结性文件

### 处理流程
```
1. 评估材料量 → 制定分批方案
   - 统计证据总量（分类计数）
   - 评估处理需求
   - 制定分批计划
   
2. 分批处理 → 生成各批次中间结果
   - 每批独立处理
   - 生成批次小结
   - 标记跨批关联
   
3. 汇总整合 → 输出统一的完整报告
   - 合并时间轴
   - 统一证据分析
   - 消除分批痕迹
```

### 质量控制
- **关联性保持**：确保关联证据不被割裂
- **一致性检查**：保持跨批次信息一致性
- **完整性验证**：最终报告不显示分批痕迹
- **准确性复核**：分批处理后进行整体复核

### 特殊注意事项
1. **合同与履约证据关联**：主合同必须与其直接相关的关键履约证据同批处理
2. **时间连续性**：同一时期的履约证据尽量同批处理
3. **法律文书优先级**：法律文书如涉及对前述证据的确认或变更，需特别标注关联关系
4. **中间结果保存**：每批处理完成后保存中间结果，便于后续汇总

**详细操作示例：** 参见分批处理机制的具体操作说明

## 📊 债权基础法律关系识别

### 一、识别标准与分类

#### 1.1 一级分类（按法律关系性质）
```
1. 合同类债权：基于双方合意产生的债权债务关系
2. 法律文书确认类债权：经生效判决书、调解书、仲裁裁决书等确认的债权
3. 法定债权：基于法律直接规定产生的债权（如税收、社保等）
```

#### 1.2 二级分类（合同类债权细分）
```
- 买卖/供应类：货物买卖、原材料供应、设备采购等
- 服务提供类：工程承包、技术服务、咨询服务、运输服务等
- 借贷类：民间借贷、企业间资金拆借等
- 租赁类：房屋租赁、设备租赁等
- 其他合同类：承揽、委托、保管等
```

### 二、识别步骤与方法

#### 2.1 证据材料清点
```
需清点的证据材料类型：
□ 合同文件（按合同名称、编号、签订日期区分）
□ 生效法律文书（按案号区分）
□ 其他证据材料（发票、收据、银行转账凭证等）

分析要点：
□ 确定每个证据材料对应的法律关系
□ 识别独立的债权产生事实
□ 避免重复计算或遗漏
```

#### 2.2 独立债权关系判断
**判断标准：** "债权基础法律关系"是指能够独立产生债权债务的法律事实
- 每一个独立的合同 = 一个独立的债权基础法律关系
- 每一份独立的生效法律文书 = 一个独立的债权基础法律关系
- 每一项无因管理/不当得利等事实 = 一个独立的债权基础法律关系

#### 2.3 识别结果输出
识别结果应包含：独立债权关系数量、一级分类、二级分类、识别清单。

**具体输出格式要求请参见《事实核查报告模板.md》第一部分：输出格式标准**

## 📋 分类查明标准

### 一、买卖/供应类合同债权查明

#### 查明要求
提供完整详细的结构化摘要，重点关注合同价款、交付验收等关键环节的约定。

**特别注意事项：**
- 当证据材料中有多份格式合同，内容几乎相同时，"查明情况"部分摘录的合同内容也应当相同
- 重点关注关于"送货单、结算单由谁签收"的条款，必须详细摘录指定签收人的姓名或职务，是判断签收单是否有效的依据
- 当合同约定的货物情况、数量同实际签收的情况不同，以送货单、结算单中实际交付验收的情况为准

#### 标准输出格式
```
根据[债权人全称]与[债务人全称]于[签订日期]签订的《[合同名称]》（编号：[编号]），债权人向债务人供应[货物简述]，合同总价款为人民币[金额]元。核心约定如下：
1. 标的物：[产品/项目名称]（见合同第[X]条）
2. 合同价款：含税总金额为人民币[金额]元（见合同第[X]条第[X]款）
3. 履行期限：[具体起止日期或履行期限要求]（见合同第[X]条）
4. 交付与验收：[交付地点、验收标准、签收要求]（见合同第[X]条）
5. 付款方式：[完整摘录付款条件关键原文，包括付款条件、付款期限、前置条件等，如"货到验收合格后7日内付款"、"收到发票后30日内付款"等]（见合同第[X]条第[X]款）
6. 违约责任：[逾期付款违约金率、逾期交货违约责任等]（见合同第[X]条）
7. 发票要求：[开票条件、开票时间要求、含税总金额等]（见合同第[X]条）
8. 争议解决：[仲裁/诉讼管辖约定]（见合同第[X]条）
9. 签收条款：[必须明确指定签收人姓名/职务，如"指定张三为唯一签收人"、"由项目经理李四签收"、"采购部门负责人签收"等，以及签收方式、签收效力等约定]（见合同第[X]条）
10. 其他关键约定：[所有权保留条款、特殊约定等]（见合同第[X]条）

**⚠️ 特别提醒：以下条款必须完整摘录原文**
- **付款条件条款**：不得概括，必须包含所有前置条件（如"货到验收合格后7日内付款"、"收到发票后30日内付款"等）
- **签收人条款**：必须明确具体人员或职务（如"指定张三为唯一签收人"、"由项目经理李四签收"等）
- **发票条款**：必须包含开票时间、金额、条件等完整要求（如"收到全部货款后5个工作日内开具增值税专用发票"等）
```

### 二、服务提供类合同债权查明

#### 查明要求
提供完整详细的结构化摘要，重点关注验收、结算等关键环节的约定。特别注意建设工程施工合同的工程价款、工期、质保金等特殊条款。

#### 标准输出格式
```
根据[债权人全称]（承包人/服务方）与[债务人全称]（发包人/委托方）于[签订日期]签订的《[合同名称]》（编号：[编号]），债权人承接[工程项目/服务内容]，合同总价款为人民币[金额]元。核心约定如下：
1. 服务内容：[具体服务项目/工程内容]（见合同第[X]条）
2. 合同价款：暂定/确定含税总金额为人民币[金额]元（见合同第[X]条第[X]款）
3. 服务期限：自[起始日期]至[结束日期]（见合同第[X]条）
4. 验收确认：[验收文件类型、签字人员要求、验收标准]（见合同第[X]条）
5. 结算确认：[结算文件要求、确认流程]（见合同第[X]条第[X]款）
6. 付款方式：[完整摘录付款条件关键原文，包括各阶段付款比例、付款条件、付款期限等，如"进度款按月结算，每月25日前支付上月完成工程量的80%"等]（见合同第[X]条第[X]款）
7. 违约责任：[逾期完工违约金、质量违约责任等]（见合同第[X]条）
8. 质保约定：[质保期限、质保金比例]（见合同第[X]条）
9. 其他关键约定：[变更、索赔、发票要求等]（见合同第[X]条）
```

### 三、借贷类债权查明

#### 查明要求
提供完整详细的结构化摘要，重点关注资金流向、利息约定、担保措施。注意区分固定利率与浮动利率，还款方式的具体约定。

#### 标准输出格式
```
根据[债权人全称]与[债务人全称]于[签订日期]签订的《[合同名称]》（编号：[编号]），债权人向债务人提供借款合计人民币[借款本金]元，约定利率为[利率标准]，借款期限自[起算日]至[到期日]。核心约定如下：
```

## 📅 债权发生情况时间轴整理

### 时间轴制作标准

#### 🚨 强制要求（绝对不可违反）

##### 【要求一】使用完整的债权发生情况格式
**时间轴中的"债权发生情况"必须采用前述"分类查明标准"中的完整格式，不得简化或概括！**
- 每个事件的描述都要包含完整的结构化信息
- 合同类要包含全部9项核心条款
- 法律文书类要完整摘录原文
- 其他证据类型按照相应的标准格式完整记录

##### 【要求二】每项都必须注明证据依据
**每个"债权发生情况"都必须明确说明对应的证据材料，不能凭空给出事实论断！**
- ❌ **错误示例**："债务人支付货款152,680.00元"（未说明依据）
- ✅ **正确做法**："根据上海农商银行收款通知（证据第X页）显示：债务人于2024-05-21支付货款152,680.00元"

##### 【要求三】不得重复或省略关键信息
**时间轴是核心输出，必须保证信息的完整性和准确性：**
- 每份合同都必须完整输出所有条款，不得用"同上"等省略表述
- 每个事件都要独立完整，便于单独查阅
- 按时间顺序严格排列，同日期事件按逻辑顺序排列

#### 基本要求
1. 时间轴应完整反映债权从产生到当前状态的全过程
2. **每个事件必须严格采用前述"分类查明标准"中对应的详细格式**
3. 合同类事件必须包含全部9项核心条款，即使某些合同条款相同也要重复完整摘录
4. 所有金额、条款引用、当事人信息必须准确完整
5. **每个事实必须有证据支撑，无证据不得认定事实**

#### 输出格式
采用表格形式，按时间顺序梳理债权发生的关键事实：

| 序号 | 日期 | 债权发生情况 |
|------|------|-------------|
| 1.   | [日期] | [直接套用前述"分类查明标准"对应格式] |
| 2.   | [日期] | [直接套用前述"分类查明标准"对应格式] |
| ...  | ...  | ... |

### 各类事件在时间轴中的描述要求

#### 重要说明
**时间轴中的每个事件都必须直接采用"分类查明标准"中相应的完整格式，不得简化。**

#### 1. 合同签订事件
直接使用对应合同类型的格式，包含完整核心条款

#### 2. 履约事件
- **发票开具**：使用发票查明的格式
- **工程单签发**：使用工程单查明的格式
- **银行转账**：使用银行转账记录的格式
- **送货单**：使用送货单的格式

#### 3. 法律文书事件
使用法律文书查明的格式，必须完整摘录原文

#### 4. 债权确认事件
使用债权确认书查明的格式

### 时间排序原则
1. 按照事件发生的时间先后顺序排列
2. 同一天发生多个事件的，按照逻辑关系排序（如：签约→履约→违约）
3. 法律文书的日期以制作日期为准
4. 履约事实以实际发生日期为准

## 🔍 证据关系综合分析

### 分析目的
在完成债权发生情况时间轴后，需要对所有证据材料进行综合分析，理清不同证据之间的效力关系、覆盖关系和引用关系，为债权分析员的深入分析提供基础。

### 一、证据效力优先级规则

#### 1.1 基本效力层级（从高到低）
```
1. 生效法律文书（判决书、调解书、仲裁裁决书等）
2. 双方共同确认文件（债权确认书、对账单、结算单等）
3. 补充协议/变更协议
4. 原始合同/协议
5. 单方制作的履约证据（送货单、发票等）
6. 单方申报材料
```

#### 1.2 时间优先原则
- **后签署文件优于先签署文件**：当存在多份同类型文件时，后签署的文件效力优先
- **最终确认文件具有总结性效力**：如结算单、对账单等总结性文件，覆盖之前的所有履约细节

#### 1.3 特殊引用规则
- **法律文书的引用回溯**：当法律文书明确引用原合同条款时（如"利息按原合同约定计算"），需回溯查看原合同相关条款
- **补充不替代原则**：补充协议仅对明确变更的条款有效，未涉及的条款仍适用原合同

### 二、综合分析要求

#### 2.1 识别证据覆盖关系
**必须明确标注以下关系：**
- 哪些原始约定被后续文件修改或替代
- 哪些履约证据被最终确认文件覆盖
- 哪些争议事项被法律文书最终确定

#### 2.2 确定最终适用条款
**对每个关键要素明确最终依据：**
- 债权金额：最终以哪份文件为准
- 利息计算：适用哪个标准（合同约定/法定/判决确定）
- 违约责任：是否有变更或司法确认
- 付款期限：最新的约定或判决要求

#### 2.3 处理复杂引用关系
**特别注意以下情形：**
1. **判决书部分引用合同**：如判决"利息按合同约定"，需明确指出回溯至原合同第X条
2. **多层修改关系**：如原合同→补充协议1→补充协议2→债权确认书，需梳理每层的修改内容
3. **交叉引用**：不同文件相互引用时，需理清最终适用逻辑

### 三、输出格式标准

```
### 证据关系综合分析（初步分析）

#### 1. 最终债权确定
基于本案全部证据材料，最终债权以[具体文件名称]为准，确定债权本金为[X]元，利息按[计算标准]计算（自[起算日期]起至[截止日期]止），其他费用[X]元。[如有特别事项说明]。

#### 2. 证据效力关系分析
根据证据效力层级原则，本案中[最高效力证据名称]具有最高法律效力，[说明原因，如"系双方共同确认文件"或"系生效法律文书"]。[次级效力证据名称]等作为基础交易证据，其中关于[具体条款]的约定已被[高效力文件]所[覆盖/修改/确认]。[如有其他证据关系说明，如引用关系、补充关系等]。

#### 3. 需要特别关注的事项
[如有需要特别提醒的事项，如：判决书引用原合同条款需回溯查看、某些证据存在瑕疵需要补充等]

#### 4. 为债权分析员提供的重点提示
[基于初步分析，指出债权分析员在进行深入分析时需要特别关注的计算要点、法律问题等]
```

### 四、分析示例

#### 示例1：存在债权确认书的情形
```
债权确认书（2023-03-10）对截至该日的欠款进行了总结确认，该确认：
- 覆盖了原合同的付款条款（原约定已无法履行）
- 确认了实际欠款金额（已扣除已付款项）
- 设定了新的还款期限

```

#### 示例2：存在生效判决书的情形
```
民事判决书（2023-10-20）对全案进行了司法确认：
- 判决主文完全替代原合同关于金额的约定
- 利息计算标准由合同约定的6%改为LPR
- 新增了迟延履行期间的加倍利息
结论：全部内容以判决书为准，原合同仅作为事实查明的参考
```

```

- 利息标准：回溯至原合同第5条，年利率6%
- 计算期间：判决确定的起止时间
结论：混合适用判决书和原合同条款
```

## ✅ 质量控制要求

### 基本质量标准
- **客观性**：确保时间、金额、当事人名称等关键信息准确无误
- **完整性**：法律文书类必须完整摘录原文，不得概括或改写
- **重点性**：合同类债权重点突出与债权金额相关的核心约定
- **中立性**：保持客观中立，仅陈述事实，不加入主观判断

### 常见错误防范
1. **混用申报材料和证据材料**：严格以证据材料为准，绝对不能依据申报表、申报书等单方面申报材料
2. **不完整摘录法律文书**：必须完整摘录，不得省略
3. **时间顺序错误**：严格按时间先后排序
4. **关键信息错误**：仔细核对金额、日期、名称等
5. **合同条款摘录不完整**：每份合同都必须完整摘录9项核心条款，不得用"条款同上"等省略表述
6. **缺少证据依据**：每个事实认定都必须注明具体的证据材料及页码

# 常见错误防范标准

## 📋 适用范围
本标准基于历史审查案例和实践经验，总结事实核查员在债权审查中的常见错误类型，提供系统性的错误防范指引。

## 🎯 防范目标
1. **提高准确性**：避免申报摘录、事实认定、证据审查等方面的错误
2. **保持客观性**：确保严格按照材料原貌记录，不添加主观判断
3. **完善流程**：通过错误防范提升整体核查质量
4. **持续改进**：基于实践反馈不断更新防范措施

## ⚠️ 申报信息摘录错误

### 错误类型1：擅自修正申报内容
**❌ 错误表现：** 对申报内容进行"修正"或"完善"
**✅ 正确做法：** 完全按照债权人申报的内容原样摘录，即使发现明显不合理之处
**📊 实际案例：** 
- 债权人未勾选债权性质，事实核查员自行判断为"普通债权"后填入金额
- 债权人勾选"职工债权"，事实核查员判断不符合条件后改为"普通债权"
**🔧 防范措施：** 
- 建立"原样摘录"检查机制
- 对任何修改行为进行标记和审查
- 仅在备注中标注"需后续核实"

### 错误类型2：申报分类自主调整
**❌ 错误表现：** 债权人将某项费用归类在"其他"，事实核查员自主移至"标准"类别
**✅ 正确做法：** 按债权人实际填写位置和分类记录
**📊 实际案例：** 债权人将迟延履行金填在"其他"类别，事实核查员自主决定移至"利息"类别
**🔧 防范措施：** 
- 严格按照申报表原始分类记录
- 不得根据"应该如何"进行调整
- 保持申报材料的原始状态

### 错误类型3：金额构成重新整理
**❌ 错误表现：** 债权人申报金额构成不清晰，事实核查员按常规分类重新整理
**✅ 正确做法：** 完全照抄债权人的原始分类和表述
**🔧 防范措施：** 
- 即使分类混乱也要原样记录
- 可在备注中说明分类问题
- 留待债权分析员进行专业判断

## 💰 证据材料审查错误

### 错误类型4：混用申报材料和证据材料
**❌ 错误表现：** 将申报表、申报书等主观陈述作为查明事实的依据
**✅ 正确做法：** 严格以证据材料（合同、判决书、发票等）为准
**📊 实际案例：** 依据申报书描述确定债权发生日期，而非依据合同签订日期
**🔧 防范措施：** 
- 建立证据材料清单
- 明确区分申报材料和证据材料
- 申报材料仅作为思考线索

### 错误类型5：合同条款摘录不完整
**❌ 错误表现：** 
- 简单概括合同内容，如"双方签订采购合同，金额100万元"
- 使用"条款同上"、"同前述合同"等省略表述
- 只摘录部分条款，遗漏其他重要条款
**✅ 正确做法：** 
- 必须按照分类查明标准输出全部9项核心条款

### 错误类型5.1：遗漏关键结算文件
**❌ 错误表现：** 未识别工程结算单、结算协议等确认最终债权金额的关键文件
**✅ 正确做法：** 必须识别所有含"结算"字样的文件，特别关注"未支付工程款金额"等关键表述
**📊 实际案例：** 第118笔债权中工程结算单含有"项目未支付工程款金额4383477"的关键信息，但查明事实完全未提及
**🔧 防范措施：** 
- 建立关键文件识别清单（含结算单、结算协议、结算书等）
- 搜索"结算"、"未支付"、"应付"等关键词
- 发现结算文件必须在时间轴中突出记录
- 每份合同都要完整摘录，即使某些条款与其他合同相同
- 每项条款都要标注在合同中的具体位置（如"见合同第X条"）
**📊 实际案例：** 
- 6份采购合同，事实核查员只详细摘录第1份，其余5份写"条款同上"
**🔧 防范措施：** 
- 使用标准化模板强制输出
- 建立条款完整性检查清单
- 对遗漏条款进行提示
- 设置"条款同上"等省略词汇的自动警告

### 错误类型6：法律文书摘录不准确
**❌ 错误表现：** 概括或改写法律文书内容
**✅ 正确做法：** 完整准确地逐字摘录原文
**📊 实际案例：** 将判决书内容概括为"判决被告支付原告100万元"
**🔧 防范措施：** 
- 使用引号标注原文部分
- 保持原文格式和标点
- 不得截取或省略

## 🔍 信息识别与判断错误

### 错误类型7：OCR识别错误忽视
**❌ 错误表现：** 未发现OCR识别的明显错误
**✅ 正确做法：** 对关键信息保持敏感，发现异常时注明"需核实原件"
**📊 实际案例：** 
- 金额"100,000"识别为"10,000"
- 身份证号码数字错位
- 日期格式识别错误
**🔧 防范措施：** 
- 对金额、日期、证件号等关键信息重点审查
- 发现逻辑异常立即标记
- 建立常见OCR错误模式库

### 错误类型8：债权关系数量识别错误
**❌ 错误表现：** 将多个独立合同认定为一个债权关系
**✅ 正确做法：** 每个独立合同/法律文书为一个独立债权关系
**📊 实际案例：** 6份采购合同被认定为1个债权关系
**🔧 防范措施：** 
- 逐一清点合同文件
- 按合同编号区分关系
- 明确列出关系清单

### 错误类型9：时间顺序排列错误
**❌ 错误表现：** 债权发生情况时间轴顺序混乱
**✅ 正确做法：** 严格按时间先后顺序排列
**🔧 防范措施：** 
- 先提取所有日期
- 排序后再整理事件
- 同日事件按逻辑排序

### 错误类型10：缺少证据依据说明
**❌ 错误表现：** 
- 直接陈述事实，不说明证据来源
- 如："债务人支付了152,680元"（未说明依据什么证据）
**✅ 正确做法：** 
- 每个事实都要注明证据依据
- 如："根据银行收款通知（证据第X页）显示，债务人支付了152,680元"
**📊 实际案例：** 
- 事实核查员写"已付款152,680元"，未说明是根据银行凭证还是其他证据
**🔧 防范措施：** 
- 强制要求每个事实陈述都以"根据XXX证据"开头
- 建立证据引用格式规范
- 对无证据支撑的事实陈述进行警告

### 错误类型11：证据关系判断错误
**❌ 错误表现：** 
- 忽视后续文件的覆盖效力，仍以原始合同为准
- 未识别法律文书的最高效力，将判决内容与合同并列
- 遗漏法律文书中的引用关系，未回溯查看原合同条款
**✅ 正确做法：** 
- 明确证据效力层级：生效法律文书>双方确认>合同协议>单方证据
- 识别覆盖关系：债权确认书覆盖之前的履约细节
- 处理引用关系：判决书引用合同条款时，需回溯原合同
**📊 实际案例：** 
- 存在债权确认书的情况下，仍按原合同计算欠款金额
- 判决书明确"利息按合同约定"，但未查看原合同利率条款
- 有补充协议修改付款期限，但仍引用原合同期限
**🔧 防范措施：** 
- 建立证据效力层级检查表
- 强制进行证据关系综合分析
- 特别标注被覆盖或修改的内容
- 对引用关系进行专门追踪

### 错误类型12：超长材料处理不当
**❌ 错误表现：** 
- 材料超过处理能力时强行一次性处理，导致遗漏
- 分批处理时割裂关联证据
- 最终报告显示分批处理痕迹
**✅ 正确做法：** 
- 识别触发条件（>100页或>50个证据项）
- 按证据类型和关联性合理分批
- 确保最终报告完整统一
**📊 实际案例：** 
- 165份证据材料强行一次处理，遗漏重要履约证据
- 将主合同与补充协议分在不同批次，导致条款理解错误
**🔧 防范措施：** 
- 建立材料量评估机制
- 制定标准分批方案
- 实施分批质量控制
- 进行最终完整性检查

## 📑 形式性核查错误

### 错误类型13：形式文件核查标准不一
**❌ 错误表现：** 对不同债权人适用不同的文件要求标准
**✅ 正确做法：** 统一适用形式性文件核查清单
**📊 实际案例：** 
- 个人债权人要求提供法人身份证明
- 企业债权人未核查营业执照
**🔧 防范措施：** 
- 区分企业/个人债权人要求
- 使用标准化核查清单
- 明确标注"无需"项目

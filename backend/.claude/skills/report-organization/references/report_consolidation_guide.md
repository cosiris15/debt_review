# Report Consolidation Guide

## Purpose

This guide provides detailed procedures for consolidating fact-checking and debt analysis technical reports into a unified review opinion form, maintaining accuracy while improving readability and professional presentation.

## Part 1: Content Extraction Principles

### Principle 1: Single Source of Truth

**Core Rule**: ALL substantive content MUST originate from the two technical reports.

**What This Means**:
- ✅ Extract facts from fact-checking report
- ✅ Extract analysis from debt analysis report
- ✅ Reorganize and reformat as needed
- ❌ Do NOT add new factual claims
- ❌ Do NOT add new analytical conclusions
- ❌ Do NOT modify calculation results
- ❌ Do NOT change legal determinations

**Rationale**: Report organizer's role is consolidation and presentation, NOT analysis.

### Principle 2: Completeness

**Core Rule**: Do NOT omit important findings or risk alerts.

**Must Include**:
- All debt relationships identified
- All amount items (本金/利息/费用)
- All calculation results
- All statute of limitations determinations
- All risk alerts or special notes
- All debt classification conclusions

**May Summarize** (but not omit):
- Lengthy evidence timelines (keep key events)
- Detailed calculation process (keep results and parameters)
- Repetitive content (consolidate intelligently)

### Principle 3: Accuracy

**Core Rule**: Do NOT modify original data or conclusions.

**Strict Preservation**:
- Amount numbers (declared and confirmed)
- Dates (bankruptcy dates, calculation dates)
- Legal conclusions (time-barred, confirmed, subordinated)
- Debt classifications (priority, ordinary, subordinated)
- Risk findings

**Formatting OK**: You can reformat presentation (tables, headings, etc.) without changing substance.

## Part 2: Content Mapping Strategy

### Understanding the Two Source Reports

#### Fact-Checking Report Contains:
- Creditor basic information
- Declaration extraction (what creditor claimed)
- Evidence classification
- Debt occurrence timeline
- Evidence hierarchy analysis
- Factual relationship establishment

#### Debt Analysis Report Contains:
- Debt relationship confirmation (based on fact-check)
- Amount item breakdown
- Interest calculation parameters and results
- Statute of limitations analysis
- Execution statute analysis (if applicable)
- Final confirmation amounts
- Debt classification (priority/ordinary/subordinated)

### Standard Content Mapping Table

| Template Section | Primary Source | Secondary Source | Notes |
|-----------------|----------------|------------------|-------|
| 债权人基本信息 | Fact-checking report | - | Name, registration, contact |
| 申报情况 | Fact-checking report | - | What creditor declared |
| 债权关系确认 | Debt analysis report | Fact-checking timeline | Independent relationships |
| 金额项目分析 | Debt analysis report | - | Breakdown and classification |
| 利息计算 | Debt analysis report | Calculation files | Results and parameters |
| 诉讼时效判断 | Debt analysis report | - | Determination and reasoning |
| 审查确认情况 | Debt analysis report | - | Final amounts by category |
| 审查意见 | Debt analysis report | - | Conclusions and recommendations |

**Flexible Adaptation**: Actual template sections may vary by client. Use template's content mapping table if provided.

### Deduplication Strategy

**Common Overlaps**:
1. **Debt relationship description**:
   - Fact-checking: Describes based on evidence
   - Debt analysis: Confirms based on fact-check
   - **Consolidation**: Use debt analysis version (it's the final determination)

2. **Amount information**:
   - Fact-checking: What creditor declared
   - Debt analysis: What's confirmed after analysis
   - **Consolidation**: Show both for comparison

3. **Evidence references**:
   - Both reports may cite same contracts/judgments
   - **Consolidation**: Cite once, reference appropriately

## Part 3: Content Reorganization Techniques

### Reorganization Principle: Conclusion → Process

**Client Reading Preference**: Typically want to see conclusion first, then supporting detail.

**Standard Reorganization**:
```
Traditional Technical Report:
1. Background
2. Evidence Review
3. Analysis Process
4. Conclusion

Client-Friendly Review Form:
1. Conclusion (确认金额)
2. Key Findings (关键发现)
3. Supporting Analysis (支持性分析)
4. Background Details (背景详情)
```

### Highlighting Key Information

**Techniques**:

1. **Executive Summary Format**:
   - Lead with final confirmation amounts
   - Highlight any【暂缓确认】items
   - Note critical risk factors

2. **Visual Emphasis**:
   - Use bold for key amounts: **确认金额: 500,000元**
   - Use tables for comparisons (declared vs. confirmed)
   - Use clear section headings

3. **Risk Flagging**:
   - Mark time-barred items clearly: **【已超过诉讼时效】**
   - Note subordinated debt: **劣后债权: XX元**
   - Highlight pending items: **【暂缓确认】**

### Professional Terminology Handling

**Principle**: Preserve legal accuracy while enhancing readability.

**Techniques**:

1. **Brief Explanations**:
   ```
   ❌ Technical: "基于（2025）沪0120民初947号民事调解书"
   ✅ Client-friendly: "基于法院调解书（（2025）沪0120民初947号）确认的债权"
   ```

2. **Consistent Terminology**:
   - Choose client's preferred terms
   - Use consistently throughout
   - Define technical terms on first use if needed

3. **Simplified Structure**:
   ```
   ❌ Technical: "诉讼时效适用三年期间，自2020年6月1日起算，期间发生一次中断..."
   ✅ Client-friendly:
   "诉讼时效判断：
   - 适用期间：3年
   - 起算日期：2020年6月1日
   - 中断事由：2022年12月15日债务人签字确认
   - 结论：未超过诉讼时效"
   ```

## Part 4: Specific Content Extraction Procedures

### Extracting Creditor Basic Information

**Source**: Fact-checking report, Section 一

**Extract**:
- Full legal name (with character-level accuracy)
- Registration number (统一社会信用代码)
- Legal representative or person in charge
- Contact information
- Company address

**Format**:
```
| 项目 | 内容 |
|------|------|
| 债权人名称 | [完整名称] |
| 统一社会信用代码 | [18位代码] |
| 法定代表人 | [姓名] |
| 联系电话 | [电话] |
| 地址 | [地址] |
```

### Extracting Declaration Summary

**Source**: Fact-checking report, Section 二

**Extract**:
- Declared total amount
- Breakdown: 本金, 利息, 其他
- Debt classification claims: 优先/普通/劣后

**Important**: Use creditor's ORIGINAL labels (don't standardize).

**Format**:
```
申报情况：
- 申报债权总额：XXX元
  - 本金：XXX元
  - 利息：XXX元
  - 其他：XXX元（包括：律师费、案件受理费等）
- 申报债权性质：普通债权
```

### Extracting Debt Relationship Confirmation

**Source**: Debt analysis report, Section 一

**Extract**:
- Number of independent debt relationships
- Specific identification of each relationship
- Legal nature (contract type, judgment, etc.)

**Format**:
```
债权基础法律关系：
- 独立关系数量：X个
- 具体清单：
  1. [关系1描述，如：XX购销合同]
  2. [关系2描述]
- 法律性质：[合同类/法律文书类]
```

### Extracting Amount Analysis

**Source**: Debt analysis report, Section 二

**Extract**:
- Amount item breakdown table
- Classification (本金/利息/费用)
- Evidence support status
- Calculation requirements

**Consolidation**: Create comparison table:
```
| 项目 | 申报金额 | 确认金额 | 性质 | 备注 |
|------|----------|----------|------|------|
| 本金 | XXX | XXX | 普通债权 | [说明] |
| 利息 | XXX | XXX | 普通/劣后 | [说明] |
| 费用 | XXX | XXX | 普通债权 | [说明] |
| 合计 | XXX | XXX | - | |
```

### Extracting Interest Calculations

**Source**: Debt analysis report, Section 四

**Extract** (Summary form):
- Calculation base
- Interest start and end dates
- Rate or standard used
- Calculation result

**Do NOT**: Copy entire detailed calculation table (that's in calculation files).

**Format**:
```
利息计算：
- 计算基数：XXX元
- 计息期间：YYYY-MM-DD 至 YYYY-MM-DD
- 适用标准：年利率X% / LPR×倍数 / 日利率万分之1.75
- 计算结果：XXX元
```

### Extracting Statute of Limitations

**Source**: Debt analysis report, Section 五

**Extract**:
- Applicable period (2-year or 3-year)
- Start date and basis
- Interruption/suspension events (if any)
- Final expiration date
- Conclusion (time-barred or not)
- Brief reasoning

**Format**:
```
诉讼时效判断：
- 起算时点：YYYY年MM月DD日（[依据]）
- 适用期间：3年
- 中断情况：[有/无，如有简述]
- 最终届满：YYYY年MM月DD日
- 时效结论：未超过诉讼时效

说理：[简要说明]
```

### Extracting Final Confirmation

**Source**: Debt analysis report, Section 七

**Extract**:
- Confirmed total amount
- Breakdown by category (本金/利息/费用)
- Breakdown by priority (优先/普通/劣后)
- Comparison with declared amounts

**Format**:
```
审查确认情况：
- 确认债权总额：XXX元
  - 本金：XXX元（普通债权）
  - 利息：XXX元（普通债权XXX元，劣后债权XXX元）
  - 费用：XXX元（普通债权）

债权分类汇总：
- 优先债权：XXX元
- 普通债权：XXX元
- 劣后债权：XXX元
```

## Part 5: Quality Control During Consolidation

### Data Consistency Verification

**Check Points**:

1. **Amount Consistency**:
   ```
   □ Declared amounts same in all mentions
   □ Confirmed amounts same in all mentions
   □ Subtotals add up to totals correctly
   □ Category breakdowns sum correctly
   ```

2. **Date Consistency**:
   ```
   □ Bankruptcy dates consistent across all mentions
   □ Interest calculation dates match source reports
   □ Statute expiration dates match calculations
   ```

3. **Classification Consistency**:
   ```
   □ Debt priority categories consistent
   □ Classifications match between detailed and summary sections
   □ Subordinated debt properly identified
   ```

### Completeness Verification

**Check Points**:
```
□ All debt relationships from analysis report included
□ All amount items from breakdown included
□ All calculation results documented
□ All statute determinations included
□ All risk alerts preserved
□ No orphaned data (mentioned once but not elsewhere)
```

### Accuracy Verification

**Check Points**:
```
□ Numbers match source reports exactly
□ No calculation errors introduced
□ Legal conclusions preserved accurately
□ Evidence citations correct (if included)
□ Creditor information accurate
```

## Part 6: Special Handling Scenarios

### Scenario 1: Multiple Debt Relationships

**Challenge**: Creditor has 3 separate contracts, each needs analysis.

**Consolidation Approach**:
- **Option A**: Separate subsections for each relationship
  ```
  ### 债权关系1：XX购销合同
  [分析]

  ### 债权关系2：YY服务合同
  [分析]

  ### 债权关系3：ZZ借款合同
  [分析]
  ```

- **Option B**: Consolidated table with all relationships
  ```
  | 关系 | 本金 | 利息 | 合计 | 性质 |
  |------|------|------|------|------|
  | 关系1 | XX | XX | XX | 普通 |
  | 关系2 | XX | XX | XX | 普通 |
  | 关系3 | XX | XX | XX | 普通 |
  | 合计 | XX | XX | XX | - |
  ```

**Choose** based on client template preference.

### Scenario 2: Time-Barred Debt

**Challenge**: Debt determined to be time-barred.

**Consolidation Approach**:
- **Clearly mark**: 【暂缓确认】in confirmation amount
- **Provide full analysis**: Include statute determination in detail
- **Explain implications**: Note that time-barred ≠ extinguished, just unenforceable

**Format**:
```
确认债权总额：【暂缓确认】

原因：经审查，该债权已超过诉讼时效期间。
- 时效起算日：YYYY-MM-DD
- 时效届满日：YYYY-MM-DD
- 债权申报日：YYYY-MM-DD

建议：因已超过诉讼时效，建议暂缓确认。
```

### Scenario 3: Calculation Errors Found

**Challenge**: During consolidation, you notice calculation in source report seems wrong.

**Handling**:
- ❌ Do NOT "fix" the calculation yourself
- ✅ Flag the issue to debt analyst for review
- ✅ Note in consolidation: "【待核实】"
- ✅ Do NOT finalize report until resolved

**Rationale**: Report organizer is NOT authorized to modify analytical conclusions.

### Scenario 4: Missing Information

**Challenge**: Template requires certain information not in source reports.

**Handling**:
- **Mark clearly**: "【信息缺失】" or "【待补充】"
- **Continue processing**: Don't halt entire consolidation
- **Note in QC**: Flag missing information
- **Escalate if critical**: If information is essential, request from prior agents

### Scenario 5: Conflicting Information

**Challenge**: Fact-checking and debt analysis reports have different amounts or conclusions.

**Resolution**:
- **Principle**: Debt analysis report is authoritative (it's the final analysis)
- **Note discrepancy** if significant
- **Use final determination** from debt analysis
- **Maintain traceability**: Keep both source reports for audit

## Part 7: Consolidation Workflow Example

### Example: Consolidating 上海金桥信息 Debt Claim

**Step 1: Extract from Fact-Checking Report**

From `上海金桥信息事实核查报告.md`:
- Creditor name: 上海金桥信息股份有限公司
- Declared: 696,733.98元
  - 工程款: 685,562.00元
  - 案件受理费: 5,328.00元
  - 迟延履行利息: 5,843.98元

**Step 2: Extract from Debt Analysis Report**

From `上海金桥信息债权分析报告.md`:
- Debt relationship: 1个（建设工程施工合同纠纷）
- Confirmed amounts:
  - 工程款本金: 685,562.00元（普通债权）

  - 迟延履行利息: 5,779.72元（劣后债权）

**Step 3: Consolidate into Review Form**

```markdown

| 项目 | 申报金额 | 确认金额 | 性质 | 说明 |
|------|----------|----------|------|------|

- 优先债权：0元

## 四、审查意见
```
